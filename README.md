# أكاديمية Full Mark - النسخة المتزامنة (Firebase)

هذه نسخة مطوّرة من نظام التسجيل الأصلي، بنفس كل الوظائف (تسجيل الطلاب، البحث، الفلترة حسب
الصف، الحسابات اليومية، إرسال البيانات عبر واتساب، الآلة الحاسبة، تسجيل الدخول)، لكن مع فرق
جوهري واحد: **البيانات الآن تُخزَّن على Firestore بدل `localStorage`**، فتصبح متزامنة تلقائيًا
ولحظيًا بين كل الأجهزة اللي بتفتح نفس الرابط.

---

## 1️⃣ خطوات الربط بـ Firebase (مرة واحدة فقط)

1. افتح [Firebase Console](https://console.firebase.google.com) → اختر مشروعك.
2. من القائمة الجانبية: **Build → Firestore Database → Create database** (لو لسه ماعملتهاش)
   - اختر أي Region قريب منك.
   - ابدأ بوضع **Test mode** مبدئيًا (تقدر تشدد القواعد لاحقًا، شوف قسم الأمان تحت).
3. من ⚙️ **Project settings → Your apps → Web app (</>)** انسخ كائن `firebaseConfig`.
4. افتح ملف `firebase-config.js` في هذا المشروع، واستبدل القيم الوهمية بالقيم الحقيقية اللي نسختها.

```js
const firebaseConfig = {
  apiKey: "...",
  authDomain: "...",
  projectId: "...",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};
```

---

## 2️⃣ نقل البيانات القديمة (بدون فقدان أي شيء)

⚠️ نفّذ هذه الخطوة **قبل** رفع المشروع الجديد بديلًا عن القديم.

1. من **نفس الجهاز والمتصفح** اللي فيه التسجيلات الحالية، افتح ملف `export-old-data.html`
   (بنفس طريقة فتح المشروع القديم، أو بسحبه وإفلاته في المتصفح).
2. اضغط زر "تنزيل نسخة احتياطية" → هينزل ملف `fullmark-backup-YYYY-MM-DD.json`.
3. بعد ما تربط `firebase-config.js` بإعداداتك الحقيقية (خطوة 1)، افتح `import-old-data.html`.
4. اختر ملف الـ JSON اللي نزّلته، ثم اضغط "رفع البيانات إلى Firestore".
5. انتظر لحد ما تشوف رسالة "اكتمل الاستيراد ✅".

بعد كده، أي جهاز يفتح المشروع هيشوف نفس البيانات القديمة + أي بيانات جديدة، كلها متزامنة.

---

## 3️⃣ رفع المشروع إلى GitHub (تستبدل بيه المشروع القديم)

استبدل كل الملفات في مستودعك الحالي على GitHub بملفات هذا المجلد:
`index.html` · `styles.css` · `script.js` · `firebase-config.js` · `Logo.png.jpeg`

الملفان `export-old-data.html` و `import-old-data.html` استخدمهم مرة واحدة فقط للترحيل،
تقدر تسيبهم في المستودع كأداة احتياطية أو تحذفهم بعد الانتهاء.

```bash
git add .
git commit -m "ربط المشروع بقاعدة بيانات Firestore لحل مشكلة عدم التزامن بين الأجهزة"
git push
```

لو المشروع مستضاف على GitHub Pages، هيتحدث تلقائيًا بعد الـ push خلال دقائق.

---

## 4️⃣ تأمين قاعدة البيانات (مهم بعد التأكد إن كل حاجة شغالة)

وضع "Test mode" يخلي أي حد على الإنترنت يقدر يقرأ/يكتب بياناتك. بعد التأكد إن التطبيق شغّال
تمام، روح لـ **Firestore Database → Rules** وحط قاعدة زي دي كحد أدنى للحماية:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /registrations/{docId} {
      allow read, write: if true; // مؤقت - يفضل ربطه بـ Firebase Authentication لاحقًا
    }
    match /app_state/{docId} {
      allow read, write: if true; // بيانات جدول الحجز (مهم! من غيرها الجدول مش بيتحفظ)
    }
  }
}
```

**ملاحظة أمان:** اسم المستخدم وكلمة المرور الحاليين (`fallmark` / ...) مكتوبين داخل الكود نفسه
اللي بيوصل لأي حد يفتح المشروع (View Source)، وده بيوفر حماية بسيطة فقط ضد المستخدم العادي،
مش حماية حقيقية. لو الموضوع يهمك، أقدر لاحقًا أربط تسجيل الدخول بـ **Firebase Authentication**
عشان يبقى فيه حماية فعلية على مستوى قاعدة البيانات نفسها، مش بس على الواجهة.

---

## 5️⃣ ملاحظة عن الصور

الصور بتتخزن حاليًا كـ Base64 داخل نفس سجل الطالب (زي ما كانت بالظبط في المشروع الأصلي).
Firestore بيحدد حجم أقصى **1 ميجابايت لكل سجل**، فلو صورة طالب كبيرة جدًا ممكن يفشل الحفظ.
لو واجهت المشكلة دي، أقدر أضيف ضغط تلقائي للصورة قبل الحفظ، أو أنقلها لـ Firebase Storage
بدل تخزينها جوه السجل مباشرة.
